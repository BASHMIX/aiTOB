⚔️ Match Lifecycle & Transitions

This state machine manages match progress. Transitions are enforced programmatically in the backend via transition_match(). 

          stateDiagram-v2
              [*] --> not_started
              
              not_started --> called : Admin Calls Match / Auto-Call
              
              called --> in_progress : Both Players Ready
              called --> complete : Auto-DQ Timer Expires (Failed Ready)
              called --> not_started : Reset/Uncall
              
              in_progress --> complete : Scores Match (AI Ref / Admin Override)
              in_progress --> conflict : Mismatched Score Report
              in_progress --> not_started : Reset
              
              conflict --> complete : Admin Resolves Dispute
              conflict --> not_started : Reset
              
              complete --> not_started : Reset Match

State Mapping & Transition Details

Status Allowed Next States Description
not_started called Initial start.gg sync state. Match is ready to be assigned to a station and called.
called in_progress, complete, not_started Summoned players to station. Creates match thread, starts 10m auto-DQ check.
in_progress complete, conflict, not_started Both checked in. Lobby active, waiting for players to report scores.
conflict complete, not_started Scores reported by players disagree. Halts auto-referee, alerts admins to resolve.
complete not_started Terminal match state. Score registered locally and reported back to start.gg.

👤 Player Registration Sequence

Bilingual Discord DM registration sequence. Tracks complete profile onboarding before validating tournament entries. 

          sequenceDiagram
              actor Player as Discord Player
              participant Bot as Discord Bot
              participant DB as SQLite Database
              participant API as FastAPI Backend

              Player->>Bot: Click "Register" in active channel
              Bot->>DB: Create player with step "startgg_linked" (unverified)
              Bot->>Player: DM Welcome Embed + start.gg OAuth login URL
              Player->>API: Link start.gg profile via Web OAuth
              API->>DB: Save startgg_id & gamer_tag
              Player->>Bot: Choose Preferred Language (EN/AR) in DMs
              Bot->>DB: Save preferred_language, advance step to "cfn_entered"
              Player->>Bot: Provide Capcom Fighter Network (CFN) ID
              Bot->>DB: Save cfn_id, advance step to "avatar_uploaded"
              Player->>Bot: Send Avatar Image file
              Bot->>API: Process avatar image size & safety
              API->>DB: Save avatar_url & mark is_verified = TRUE
              Bot->>Player: Send Registration Confirmation DM

Registration Step Sequence

Step Name Database Representation Action Description
1. Link Start.gg registration_step = 'startgg_linked' Redirects user to the FastAPI endpoint to associate start.gg profile.
2. Select Language preferred_language = 'ar' / 'en' Saves user choice for customized messages in EN or AR.
3. Input CFN ID cfn_id = 'input_value' Associates user CFN ID for in-game lobbies and overlay visual streams.
4. Upload Avatar avatar_path = 'saved_path' Player uploads image. Backend checks safety, processes size, updates field.
5. Verified is_verified = TRUE Saves record. Verified players can be checked into active brackets.
